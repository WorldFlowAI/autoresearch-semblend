\begin{abstract}
Transformer KV cache systems such as LMCache and vLLM prefix caching
require \emph{exact token-prefix matching}, missing semantically
equivalent prompts that differ in wording or sentence ordering. We
present \semblend, an in-process, GPU-native semantic KV cache for
vLLM that discovers donor prompts via sentence-embedding cosine search
and injects their cached KV tensors with \textbf{exact RoPE delta
correction}---exploiting the rotation group structure
$K_\text{corrected} = \text{RoPE}(\Delta) \times K_\text{donor}$
to correct donor positions at $O(Nd)$ cost per layer.
A calibrated bathtub recomputation schedule selectively recomputes
high-drift early/late layers, with optional GPU-accelerated CAGRA search
for large donor pools.

Evaluated on five NLP benchmarks (CNN/DailyMail, XSum, MultiNews,
WikiHow, SAMSum) across Qwen2.5-7B-AWQ and LLaMA-3.1-8B-AWQ on A10G,
\semblend achieves $\mathbf{2.2\times}$--$\mathbf{{\sim}12\times}$ TTFT
speedup at 2K--32K tokens (hit-only, single-donor synthetic context). On datasets with ${\geq}88\%$ hit rate
(WikiHow, XSum, synthetic), PPL ratio stays ${\leq}1.065$ on both
models. Elevated PPL on lower-hit datasets (SAMSum, CNN/DM) is driven
by miss-run outliers, not KV injection. On 400 real WildChat conversation pairs (250 short $+$ 150 long),
\semblend achieves 30\% hit rate with $1.63$--$1.88\times$ hit-only
speedup and no model modifications.
Enterprise workload evaluation on template-based reports,
customer support tickets, multi-tenant APIs, and cross-instruction RAG
shows 100\% hit rates and $2.5$--$4.2\times$ speedup on
template-dominated patterns, confirming zero-configuration
applicability to production LLM deployments.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Large language model (LLM) inference is dominated by the
\emph{prefill phase}---computing key-value (KV) tensors for the input
prompt before autoregressive decoding begins. For a 32-layer
transformer processing 512 tokens with 8~KV heads and
128-dimensional head size, prefill produces ${\sim}512$\,MB of FP16 KV
state:
\begin{equation}
  \text{KV bytes} = 2 \times L \times H \times T \times D \times 2
  \label{eq:kv-size}
\end{equation}
where $L{=}32$ layers, $H{=}8$ heads, $T{=}512$ tokens, $D{=}128$ head
dimension. Prefill cost scales superlinearly with prompt length.

Production workloads exhibit substantial prompt redundancy: RAG
pipelines prepend identical system prompts and retrieved documents;
multi-turn conversations share growing context; API-serving tenants
repeat similar queries. Existing systems---PagedAttention~\cite{pagedattention},
vLLM prefix caching, LMCache~\cite{lmcache}---exploit this through
\emph{exact} token prefix matching. LMCache further extends this with
distributed P2P KV transfer across inference instances, but retains the
fundamental constraint: tokens must match exactly for cache reuse.

However, exact matching misses a common production pattern:
\emph{semantically similar prompts with different wording}. ``What is
machine learning?'' and ``Can you tell me about machine learning?''
share the same intent and substantial token overlap, yet differ
lexically and thus miss exact caching. SemShareKV~\cite{semsharekv}
established that cross-prompt KV reuse is feasible using token-level
locality-sensitive hashing (LSH), achieving $6.25\times$ speedup at
5K tokens. \semblend builds on this insight, advancing it along three
dimensions: (1)~\emph{in-process production integration} with vLLM via
a drop-in KV connector---no model modification, no proxy---with ${\sim}8$\,ms
overhead vs.\ SemShareKV's external-service model; (2)~\emph{exact RoPE
delta correction} that eliminates position-encoding error for non-contiguous
reuse (SemShareKV relies on first-layer full recomputation as a heuristic);
and (3)~\emph{architecture-specific calibrated recomputation} that accounts
for LLaMA (inverted-funnel) vs.\ Qwen (funnel) layer sensitivities, validated
empirically on both architectures across five real-world datasets.
\semblend extends LMCache's architecture to cover the cases that
exact-match caching misses: when no token-prefix match exists,
\semblend performs embedding-level semantic search to find a \emph{donor}
prompt whose cached KV tensors may be reusable.

\paragraph{Positioning relative to LMCache.}
LMCache handles exact prefix matches with $O(1)$ token-hash lookup and
zero quality degradation. \semblend is \emph{complementary}: it activates
only when LMCache's exact-match path misses, extending the cache's
coverage to semantically similar prompts. The in-process architecture
adds only ${\sim}8$\,ms overhead (vs.\ ${\sim}150$\,ms for a remote
embedding gateway), making this extension viable even for small models.
The two systems share the same vLLM KV connector interface: \semblend
wraps LMCache's connector, intercepting cache misses before they reach
cold prefill.

\paragraph{Contributions.}
We make five contributions:
\begin{enumerate}[leftmargin=*,nosep]
\item \textbf{In-process donor discovery} (\S\ref{sec:pipeline}): an
  in-process architecture achieving ${\sim}8$\,ms total overhead
  (embedding + search + alignment) with zero network hops, using a
  CPU sentence encoder, numpy cosine matching, and compiled token
  alignment---making semantic KV search viable for any model size.

\item \textbf{Scalability path}: the same architecture extends to
  GPU-accelerated CAGRA ANN search (\S\ref{sec:scaled}) for pools
  exceeding 100K donors, achieving $\mathbf{1.37}$\,ms constant
  latency at $N{=}1$\,M donors.

\item \textbf{Architecture-specific calibrated quality model}
  (\S\ref{sec:recomp}): a bathtub-curve layer sensitivity function
  with per-family presets---LLaMA (inverted funnel: early layers
  recomputed, $\sigma_e{>}\sigma_l$) and Qwen (funnel: late layers
  recomputed, $\sigma_l{>}\sigma_e$)---consistent with kv-cache-physics
  findings~\cite{kvphysics}, validated against published data points
  from CacheBlend, KVShare, ProphetKV, and SemShareKV.

\item \textbf{Exact RoPE delta correction} (\S\ref{sec:rope-correction}):
  a mathematically exact method for correcting donor K tensors to
  arbitrary target positions by exploiting the rotation group structure
  of RoPE, enabling non-contiguous KV reuse (REORDER, PARTIAL,
  paraphrase) at $O(Nd)$ cost per layer---negligible compared to the
  $O(N^2 d)$ attention it replaces.

\item \textbf{Empirical evaluation} on two model architectures
  (Qwen2.5-7B-AWQ and LLaMA-3.1-8B-AWQ) across five real-world NLP
  datasets, demonstrating $2.2\times$--$\mathbf{{\sim}12\times}$ TTFT speedup at
  2K--32K tokens with quality preserved (PPL ratio ${\leq}1.065$ on
  high-hit datasets), plus architectural analysis
  of the prefix cache coverage gap (\S\ref{sec:eval}).
\end{enumerate}

Table~\ref{tab:taxonomy} summarizes the landscape. We identify four
categories of cache reuse:

\textbf{Response-level systems.}
GPTCache~\cite{gptcache} and LangChain's semantic cache~\cite{langchain}
intercept queries upstream of the LLM and return cached responses when
embedding similarity exceeds a threshold. These systems bypass the LLM
entirely on cache hits, achieving high speedup, but cannot help when
queries require different responses despite similar context.

\textbf{Exact-prefix KV systems.}
CacheBlend~\cite{cacheblend} introduced selective layer recomputation
based on KV deviation scores but is limited to exact prefix matches,
achieving 2.2--3.3$\times$ TTFT reduction only when prefixes align.
LMCache~\cite{lmcache} adds distributed P2P transfer but retains the
exact-match constraint.

\textbf{Token-level semantic KV systems.}
KVShare~\cite{kvshare} uses Divergence-based Hit Detection (DHD)
for multi-tenant serving, claiming 9.39$\times$ TTFT reduction on
semantic hits. SemShareKV~\cite{semsharekv} is the closest prior work:
it demonstrates that prompts sharing semantically similar content can
reuse each other's KV tensors via token-level locality-sensitive hashing
(LSH), achieving $6.25\times$ speedup at 5K tokens. \semblend builds
directly on SemShareKV's key insight while advancing three aspects:
(i)~\emph{in-process vLLM integration} via the standard KV connector
interface (zero model modification, zero proxy, ${\sim}8$\,ms overhead vs.\
SemShareKV's $O(T)$ LSH requiring a separate lookup pass); (ii)~\emph{exact
RoPE delta correction} exploiting the rotation group property to correct
donor K tensors at $O(Nd)$ cost per layer, eliminating SemShareKV's
reliance on first-layer full recomputation as a positional heuristic;
and (iii)~\emph{architecture-specific bathtub calibration} for LLaMA
and Qwen family models, validated with empirical quality measurements
across five real-world datasets. SemShareKV's $O(T)$ overhead provides
minimal benefit below 700 tokens~\cite{semsharekv}; \semblend's ${\sim}8$\,ms
constant overhead extends viability to any model at any prompt length
where cold prefill exceeds ${\sim}50$\,ms.
KVCOMM~\cite{kvcomm} addresses cross-agent KV reuse in multi-agent
systems where donor and target sequences have different RoPE offsets;
it approximates position correction via a pool of precomputed
``anchor'' embeddings (interpolation). \semblend uses exact per-chunk
delta correction---no fine-tuned anchors required.
SentenceKV~\cite{sentencekv} groups tokens at sentence level and
offloads semantically similar KV entries to CPU for within-request
compression, orthogonal to \semblend's cross-request donor injection.

\textbf{RAG-specific KV systems.}
ProphetKV~\cite{prophetkv} uses a query-attention pass against cached
keys to score per-token relevance, selectively recomputing the
top-$p$ attention-critical tokens while reusing the rest---achieving
96--101\% of full-prefill quality at 20\% recomputation budget and
4.6--4.7$\times$ TTFT speedup at 16K context (H100). ProphetKV
addresses \emph{intra-request} cross-chunk attention (retrieved
documents do not attend to each other in a pre-computed cache),
a problem orthogonal to \semblend's cross-request donor reuse.
KVLink~\cite{kvlink} precomputes
per-document KV caches for RAG scenarios and concatenates them with
positional embedding adjustment, achieving up to 96\% TTFT reduction.
Unlike \semblend, KVLink requires model fine-tuning (trainable special
tokens + mixed-data fine-tuning) and is designed for static document
retrieval rather than semantic query matching.
EPIC~\cite{epic} uses semantic chunking (KVSplit) and attention-sink
correction (AttnLink) to reuse KV chunks at arbitrary absolute
positions in RAG, achieving up to 96\% TTFT reduction on identical
retrieved documents. EPIC handles position shift for \emph{identical}
content; \semblend extends this to \emph{semantically similar} content
via embedding-level donor discovery.

\textbf{Layer criticality in KV reuse.}
DroidSpeak~\cite{droidspeak} profiles transformer layer criticality
for KV cache sharing across heterogeneous LLMs, finding that only
${\sim}11\%$ of layers are critical for output quality.
KV Cache Physics~\cite{kvphysics} identifies architecture-specific
information flow: LLaMA exhibits an ``inverted funnel'' (early layers
1--8 consolidate; late layers are redundant), while Qwen exhibits a
``funnel'' (late layers 20--28 consolidate; early layers more redundant).
Our calibrated bathtub curve (\S\ref{sec:recomp}) directly implements
the architecture-specific insight: LLaMA parameters ($\sigma_e{=}0.45$,
$\sigma_l{=}0.15$) recompute early layers $[0,1,2]$ (9.4\%), targeting
the inverted-funnel consolidation point; Qwen parameters
($\sigma_e{=}0.15$, $\sigma_l{=}0.35$) recompute late layers
$[26,27]$ (7.1\%), targeting the funnel convergence layers.
This architecture-aware selection is consistent with published
per-layer KV analysis~\cite{kvphysics,droidspeak} and validated
indirectly by our PPL measurements (1.001--1.065 across five
datasets and both models), which confirm the architecture-aware
calibration is near-optimal in practice.
Poudel~\cite{statefulkv} provides complementary evidence: in
stateful multi-turn KV eviction, even high-retention strategies
(${\geq}99\%$ via AttentionTop) cause degenerative output if
they scramble the positional signal---independently validating
the importance of exact RoPE position correction for any system
that perturbs KV cache positions.
MEPIC~\cite{mepic} addresses the same RoPE challenge by storing KV
without pre-applied position encoding (NoPE format) and injecting
positions via a fused CUDA kernel at inference time---elegant but
requires a custom inference kernel and modified vLLM internals.
We implement the NoPE two-step as an optional mode in \semblend
(\texttt{SEMBLEND\_USE\_NOPE=1}): strip~$\mathbf{K}_\text{raw}{=}\mathrm{RoPE}({-}p_s){\times}\mathbf{K}$,
then inject~$\mathbf{K}_\text{tgt}{=}\mathrm{RoPE}(p_t){\times}\mathbf{K}_\text{raw}$.
By the RoPE group property $\mathrm{RoPE}(a){\times}\mathrm{RoPE}(-b){=}\mathrm{RoPE}(a{-}b)$,
NoPE and delta correction produce \emph{identical} $K_\text{tgt}$---confirmed
empirically across all six benchmark datasets with
identical PPL ratio and TTFT speedup under both modes.
\semblend defaults to delta correction (no storage format change
required), with NoPE available as a validated alternative.

\semblend operates at the \emph{embedding level}: a single sentence
embedding replaces per-token matching. The in-process tier uses numpy
brute-force ($O(N)$, ${\leq}0.14$\,ms for $N{\leq}10\text{K}$); for
larger pools, the optional cuVS brute-force (${\leq}$0.71\,ms up to 100K) or
CAGRA ANN ($\mathbf{1.37}$\,ms constant up to 1M donors) can be used.
Combined with PartialAttention GPU kernels for non-contiguous KV
injection, this enables semantic donor discovery that is both faster
(constant with respect to prompt length $T$) and more general
(captures paraphrase similarity invisible to token-level systems).

\subsection{In-Process Donor Discovery}
\label{sec:design:embedding}
\label{sec:pipeline}

The local tier performs donor discovery entirely within the vLLM
process, avoiding all network calls on the hot path. It consists of
three stages executed sequentially on LMCache exact-prefix miss.

\paragraph{Stage 1: Sentence Embedding (${\sim}5$\,ms).}
\semblend computes a sentence embedding $e_q \in \mathbb{R}^{384}$
using all-MiniLM-L6-v2~\cite{minilm} on CPU. This 80\,MB model loads
in ${\sim}2$\,s at startup and embeds a prompt in ${\sim}5$\,ms on a
single CPU core---30$\times$ faster than a remote jina-embeddings-v4
call (${\sim}150$\,ms including HTTP overhead). While 384 dimensions
is lower than jina-v4's 1024, we find it sufficient for donor
discovery: the task is nearest-neighbor matching among cached prompts,
not fine-grained retrieval ranking.

For long prompts ($T > 2{,}000$ tokens), MiniLM's 512-token input
limit means that head-only truncation would see only the first
${\sim}12\%$ of a 16K document, degrading recall for content-heavy
RAG prompts. \semblend instead decodes a 2{,}000-token
sample spanning the beginning (40\%), center (30\%), and end (30\%)
of the token sequence, then feeds the decoded \emph{text} to MiniLM,
which internally truncates to its 512-token context window.
The 40/30/30 split over-weights the head (which typically contains
the instruction and topic sentence) while ensuring body and conclusion
coverage; the exact ratios are not critical as long as all three
segments are represented. This sampling is applied symmetrically
at query time and donor registration.
Empirically, this improves partial-content (75\% overlap) hit rate
from 0\% to 87.5\% at 8K tokens, with $1.95\times$ speedup,
demonstrating that coverage of body content---not just the instruction
prefix---is critical for donor recall.

\paragraph{Stage 2: Numpy Cosine Search (${\sim}1$\,ms).}
For $N \leq 10\text{K}$ donors, brute-force cosine similarity via
a single matrix-vector multiply is optimal:
\begin{equation}
  s_i = \frac{e_q \cdot e_i}{\|e_q\| \|e_i\|}, \quad i \in [1, N]
  \label{eq:cosine-local}
\end{equation}
With pre-normalized embeddings (unit vectors), this reduces to
$s = E \cdot e_q$ where $E \in \mathbb{R}^{N \times d}$---a single
BLAS call completing in $0.14$\,ms for $N{=}10\text{K}$, $d{=}384$
(15\,MB of embeddings; empirically measured). No index construction is needed. An LRU
eviction policy bounds memory at the configured maximum.

\paragraph{Stage 3: Compiled Token Alignment ($<$1\,ms).}
For the top donor, \semblend computes token-level alignment using
rapidfuzz's~\cite{rapidfuzz} compiled C++ Levenshtein
implementation---$<$1\,ms for 8K-token prompts versus
${\sim}2\text{--}4$\,s for pure Python Wagner-Fischer~\cite{wagnerfischer}.
The alignment produces per-position classifications:
\emph{copy\_from\_donor} (reuse KV) or \emph{placeholder} (recompute).
Positions are aligned to LMCache's 256-token chunk boundaries:
\begin{equation}
  N_\text{reuse} = \lfloor N_\text{matched} / 256 \rfloor \times 256
  \label{eq:chunk-align}
\end{equation}

\paragraph{KV Injection.}
On donor match, \semblend monkeypatches LMCache's internal lookup to
return the donor's chunk hash, causing \texttt{start\_load\_kv()} to
load the donor's KV tensors from LMCache's CPU offload store. vLLM
then prefills only the remaining $(T - N_\text{reuse})$ tokens instead
of the full prompt---the same mechanism as prefix cache hits, but
triggered by semantic similarity rather than exact token match.

The in-process pipeline adds only ${\sim}8$\,ms overhead (vs.\ ${\sim}150$\,ms
for a remote embedding gateway), making semantic KV search viable for
prompts as short as ${\sim}100$ tokens on models as small as 1.5B parameters.

\subsection{Scalability: GPU-Accelerated ANN Search}
\label{sec:scaled}

For deployments with $>$100K unique prompt patterns or multiple vLLM
instances, \semblend can optionally use CAGRA~\cite{cagra}
GPU-accelerated approximate nearest neighbor search via the Synapse
gateway, providing $O(\log N)$ search up to $N{=}1\text{M}$ donors
(1.37\,ms constant, A10G).

\paragraph{Architecture.}
Each vLLM instance publishes donor metadata (embedding, token count,
chunk hash) to the gateway on \texttt{save\_kv\_layer()}. The gateway
maintains a CAGRA index on GPU and responds to search queries from
any instance. On a CAGRA hit, the gateway returns the donor's source
instance and chunk hash; the requesting instance retrieves the donor's
KV via LMCache's existing P2P transfer mechanism.

\paragraph{Latency.}
The optional CAGRA tier adds ${\sim}50$\,ms (gateway HTTP + CAGRA search
+ response). This is invoked \emph{asynchronously} on in-process miss:
the current request proceeds with cold prefill while the CAGRA
result pre-warms the local donor store for future similar queries.

\paragraph{Complexity.}
CAGRA search is $O(\log N)$ and independent of prompt length $T$---a
key advantage over SemShareKV's~\cite{semsharekv} $O(T)$ token-level
LSH. Empirically, CAGRA achieves $\mathbf{1.37}$\,ms \emph{constant}
from $N{=}100$ to $N{=}1\text{M}$ donors on a dedicated A10G
(build time 7.3\,s one-time); cuVS brute-force completes in 0.71\,ms
at $N{=}100\text{K}$ and is optimal for co-located
deployments below that threshold.

\paragraph{Fleet-scale deployment.}
On an 8-worker fleet (LLaMA-3.1-8B-AWQ-INT4) with controlled
exact-match pairs, queries route to a semantically matching worker
achieving $2.6\times$ TTFT speedup; at 16K tokens, $3.8\times$
($3{,}222\,\text{ms} \to 850\,\text{ms}$) with negligible routing overhead.

\subsection{Token Alignment}
\label{sec:alignment}

All discovery paths use the same alignment algorithm. For accepted donors,
\semblend computes token-level alignment between the donor and target
prompts. The alignment produces per-position classifications:
\emph{copy\_from\_donor} (token matches, reuse KV) or
\emph{placeholder} (token diverges, recompute). This alignment is
encoded as a transfer plan with explicit donor$\to$target position
mappings.

\begin{algorithm}[t]
\caption{Donor Discovery}
\label{alg:delta}
\begin{algorithmic}[1]
\Require Query embedding $e_q$, local store $\mathcal{L}$, threshold
  $s_{\min}$
\Ensure Transfer plan $P$ or $\bot$ (miss)
\State $\mathcal{D}_1 \gets \text{NumpyCosine}(\mathcal{L}, e_q, k)$
  \Comment{$O(N)$, ${\sim}1$\,ms}
\For{$(d, s_d) \in \mathcal{D}_1$ by descending $s_d$}
  \If{$s_d < s_{\min}$} \textbf{break} \EndIf
  \State $A \gets \text{rapidfuzz}(d.\text{tokens}, q.\text{tokens})$
    \Comment{${<}1$\,ms}
  \State $P \gets \text{BuildTransferPlan}(A)$
  \If{$P.\text{reuse\_ratio} \geq 0.5$}
    \State \Return $P$
  \EndIf
\EndFor
\State \textbf{async:} query optional CAGRA gateway \Comment{non-blocking}
\State \Return $\bot$
\end{algorithmic}
\end{algorithm}

Algorithm~\ref{alg:delta} shows the complete donor discovery pipeline.
The in-process store is always checked first; on miss, the optional
CAGRA gateway is queried asynchronously to pre-warm the local store
for future similar queries.

\subsection{Calibrated Layer-Selective Recomputation}
\label{sec:recomp}

Each transformer layer $\ell$ has a deviation sensitivity following a
\textbf{bathtub curve}:
\begin{equation}
  \sigma(\ell) = \sigma_\text{base} + \sigma_e \cdot e^{-\ell/\tau_e} + \sigma_l \cdot e^{-(L-\ell)/\tau_l}
  \label{eq:bathtub}
\end{equation}
where $\sigma_e$, $\sigma_l$ are early/late layer bump magnitudes,
$\tau_e$, $\tau_l$ are decay constants, and $L$ is the total layer
count. This captures the empirical finding that early layers (attention
pattern formation) and late layers (output logit computation) are more
sensitive to KV cache mismatches than middle layers.

\paragraph{Recomputation decision.} Layer $\ell$ is recomputed if
$\sigma(\ell) \cdot m > \theta$, where $m = 1 - \text{overlap}(q,d)$ is
the token mismatch fraction and $\theta = 0.3$ is the deviation threshold.
With architecture-specific parameters ($\sigma_\text{base}{=}0.12$),
LLaMA uses ($\sigma_e{=}0.45$, $\tau_e{=}2.5$, $\sigma_l{=}0.15$,
$\tau_l{=}4.0$) recomputing early layers $[0,1,2]$ (inverted-funnel),
while Qwen uses ($\sigma_e{=}0.15$, $\tau_e{=}3.0$, $\sigma_l{=}0.35$,
$\tau_l{=}3.0$) recomputing late layers $[26,27]$ (funnel). The middle
layers form a robust trough below threshold in both cases.

\subsection{PartialAttention Mechanism}
\label{sec:partial}

When a semantic donor matches with non-contiguous aligned positions
(e.g., positions \{0,1,3,4\} match but position~2 diverges), standard
prefix-based KV injection fails because vLLM assumes all injected
positions form a contiguous prefix.

\semblend defines four attention modes per position per layer:

\begin{description}[leftmargin=*,nosep]
\item[FULL] Compute Q,K,V from scratch---standard prefill.
\item[REUSE] Reuse cached K,V from donor---no computation needed.
\item[PARTIAL] Compute fresh Q,K,V but cross-attend to all cached K,V.
\item[SKIP\_LAYER] Skip the entire layer (when layer deviation is below
  threshold, per CacheBlend's insight).
\end{description}

\begin{algorithm}[t]
\caption{Build PartialAttention Plan}
\label{alg:partial}
\begin{algorithmic}[1]
\Require Transfer plan $P$ with slot actions, layer hints
\Ensure PartialAttentionPlan with per-layer masks
\For{each slot action $a$ in $P$}
  \If{$a.\text{action} = \text{copy\_from\_donor}$}
    \State $\text{mask}[a.\text{target\_pos}] \gets \text{REUSE}(a.\text{donor\_pos})$
  \Else
    \State $\text{mask}[a.\text{target\_pos}] \gets \text{PARTIAL}$
  \EndIf
\EndFor
\For{each layer $\ell$ in $[0, L)$}
  \If{layer\_hint[$\ell$].recompute\_all}
    \State $\text{layer\_mask}[\ell] \gets \text{all FULL}$
  \Else
    \State $\text{layer\_mask}[\ell] \gets \text{per-position masks}$
  \EndIf
\EndFor
\State \Return plan with layer masks, computation ratio
\end{algorithmic}
\end{algorithm}

The computation ratio quantifies the fraction of total FLOPs compared
to full prefill:
\begin{equation}
  C_\text{ratio} = \frac{L_\text{partial} \cdot N_\text{placeholder} + L_\text{full} \cdot T}{L \cdot T}
  \label{eq:cratio}
\end{equation}
where $L_\text{partial}$ is the count of non-recompute layers,
$N_\text{placeholder}$ is the number of placeholder positions,
$L_\text{full}$ is the count of full-recompute layers, $T$ is the target
length, and $L$ is the total layers.

\subsubsection{Triton GPU Kernel Implementation}
\label{sec:triton}

PartialAttention is implemented as a three-stage Triton~\cite{triton}
GPU pipeline: (1)~\texttt{scatter\_donor\_kv} scatters donor K/V
tensors to target positions across all layers and heads in parallel;
(2)~\texttt{masked\_qkv\_proj} applies the QKV projection only at
placeholder positions via tiled GEMM with per-row masking; and
(3)~\texttt{partial\_prefill\_attn} computes causal attention at
compute positions using the online softmax algorithm~\cite{flashattention},
attending over the merged donor+fresh KV cache. The three kernels are chained in a single CUDA stream with a PyTorch
CPU fallback. Table~\ref{app:tab:kernels} summarises grid and block

\subsection{RoPE Delta Correction for Non-Contiguous KV Reuse}
\label{sec:rope-correction}

A key challenge for semantic KV reuse arises from Rotary Position
Embeddings (RoPE)~\cite{rope}, used by virtually all modern transformers
including LLaMA, Qwen, Mistral, and Gemma. RoPE applies position-dependent
rotation to Q and K \emph{before} KV caching:
\begin{equation}
  q_i, k_i = \text{RoPE}(i, W_Q x_i), \text{RoPE}(i, W_K x_i)
  \label{eq:rope}
\end{equation}
where $i$ is the absolute position index. The cached K tensor at position
$i$ has the rotation for position $i$ \emph{baked in}. If a donor cached
K at position $d$ and the target needs it at position $t$, na\"{\i}ve
reuse produces incorrect attention because
$\text{RoPE}(d, k) \neq \text{RoPE}(t, k)$.

\paragraph{Exact delta correction.}
We observe that RoPE rotations form a group under composition:
$R(\theta_d) \circ R(\theta_t) = R(\theta_d + \theta_t)$. This means
we can \emph{exactly} correct a donor K cached at position $d$ for use
at target position $t$ by applying a single rotation by the position
delta $\Delta = t - d$:
\begin{equation}
  K_\text{corrected} = \text{RoPE}(\Delta, K_\text{donor})
  \label{eq:rope-correction}
\end{equation}
This works because $\text{RoPE}(t, k) = \text{RoPE}(\Delta) \circ
\text{RoPE}(d, k)$---the composition of the donor's baked-in rotation
with the delta rotation produces the exact rotation needed at the target
position. The correction is \emph{mathematically exact}: it introduces
zero approximation error. The V cache requires no correction, as RoPE
is applied only to Q and K.

\paragraph{Empirical validation ($\Delta{\neq}0$ micro-benchmark).}
We validate the correction directly by generating K tensors via
standard RoPE at source positions $[0{\ldots}L{-}1]$, applying
\texttt{rope\_correct\_k} to shift to target positions
$[\Delta{\ldots}\Delta{+}L{-}1]$, and comparing against ground-truth K
tensors computed natively at target positions. Across $\Delta \in
\{0, 1, 16, 32, 64, 128, 256, 512, 1024, 2048\}$ with
$L{=}512$ ($n_\text{heads}{=}8$, $d{=}128$, float32), the corrected
relative L2 error is $< 10^{-5}$ at all $\Delta$ values---a
$31\,000$--$35\,000{\times}$ reduction over na\"{\i}ve donor reuse
without correction. At 8K tokens ($L{=}8192$) with $\Delta{=}2048$,
correction reduces L2 error from 105.6 to $< 0.006$---a $35\,000\times$
improvement. All $\Delta$ values pass the $10^{-4}$ relative error
threshold (float32 precision ceiling). End-to-end validation under
realistic deployment conditions confirms these micro-benchmark results
translate to 100\% quality recovery (\S\ref{sec:analysis:rope}).

\paragraph{Computational cost.}
For each reused K position, the correction requires one complex
multiplication per head dimension pair:
\begin{align}
  k'_{2j}   &= k_{2j} \cos\theta_\Delta - k_{2j+1} \sin\theta_\Delta \nonumber \\
  k'_{2j+1} &= k_{2j} \sin\theta_\Delta + k_{2j+1} \cos\theta_\Delta
  \label{eq:rope-complex}
\end{align}
where $\theta_\Delta = \Delta \cdot \theta_j$ and
$\theta_j = 10000^{-2j/d}$ is the RoPE frequency for dimension pair $j$.
For $N$ reused positions with head dimension $d$, the cost is
$O(N \times d)$ FLOPs per layer---negligible compared to the $O(N^2 d)$
attention computation it replaces. On an A10G GPU, correcting 8K positions
takes ${\sim}7\,\mu\text{s}$ per layer (measured via CUDA events).

\paragraph{Implementation.}
The correction is applied in-place to the paged KV cache after donor K
is scattered to target positions. A position map $\{(d_i, t_i)\}$,
built from the alignment slot actions
(\S\ref{sec:alignment}), identifies which positions need correction
(only those where $d_i \neq t_i$). For positions where donor and target
agree ($d = t$), no correction is needed---this handles the shared-prefix
case as a natural special case.

\paragraph{Complementary to bathtub recomputation.}
RoPE correction fixes \emph{position encoding} errors but not
\emph{attention pattern drift} from context differences. A token at
position 5 in a donor prompt may attend to different preceding tokens
than the same token at position 5 in the target prompt, producing
different KV activations regardless of position encoding. The calibrated
bathtub curve (\S\ref{sec:recomp}) addresses this orthogonal concern by
identifying layers where context-dependent attention drift is high
(early and late layers) and selectively recomputing them, while
reusing RoPE-corrected donor KV for the stable middle layers.

\paragraph{Enabling non-contiguous reuse.}
With delta correction, all three semantic reuse scenarios become viable:
\begin{enumerate}[leftmargin=*,nosep]
\item \textbf{REORDER}: donor tokens at positions $[0,1,2,3]$ reused
  at target positions $[2,3,0,1]$ with $\Delta = [+2,+2,-2,-2]$
  corrections.
\item \textbf{PARTIAL overlap}: shared tokens at different offsets
  corrected by their respective deltas.
\item \textbf{Paraphrase}: semantically equivalent tokens at different
  positions receive appropriate position corrections after alignment.
\end{enumerate}
This transforms semantic KV reuse from prefix-only to fully
non-contiguous, unlocking the PartialAttention kernels
(\S\ref{sec:partial}) for all reuse scenarios on RoPE-based models.

\end{itemize}

\paragraph{Donor discovery.}
\semblend: in-process MiniLM-L6-v2
(384-dim, ${\sim}5$\,ms on CPU), numpy cosine donor store,
rapidfuzz compiled alignment, calibrated bathtub recomputation.
Total pipeline overhead: ${\sim}8$\,ms with zero network hops.

\paragraph{Dataset.} We construct controlled RAG
scenarios using context chunks drawn from five established
NLP benchmarks used by prior work~\cite{semsharekv}: CNN/DailyMail,
XSum, MultiNews, WikiHow, and SAMSum. Chunks
can be reordered, removed, or replaced to produce REORDER, PARTIAL,
and COLD test cases with known overlap characteristics.

\paragraph{Protocol.} We restart the inference server before each
prompt length, clearing both the in-memory donor store and LMCache's
CPU KV buffer to eliminate cross-length contamination.
Within each length, execution is two-phase:
(1)~\emph{Seed}---send seed prompts to populate the donor store, then
(2)~\emph{Test}---send variant prompts with unique reference IDs
(preventing prefix cache hits), measuring per-query TTFT via SSE
streaming. Cold baselines use a fresh server state with no donors.
Synthetic-context experiments use $n{=}5$ unique contexts per length;
cluster-based cross-dataset experiments use $n{=}8$ per length,
drawn from independent cluster files (no context reuse across lengths).

\paragraph{Hit rate definition.}
Throughout this paper, \textbf{hit rate} denotes the fraction of
queries for which SemBlend finds a donor with cosine similarity
${\geq}\tau$ (default $\tau{=}0.60$) \emph{and} successfully loads
at least one LMCache chunk from that donor's KV. A query with a
semantic match but evicted KV data counts as a miss (the
multi-candidate fallback, \S\ref{sec:semblend-injection-results},
mitigates this). Hit-only metrics (speedup, PPL) are conditioned
on this definition.

\paragraph{Statistical methodology.} We report P50 TTFT with the
range of individual measurements, with per-length server restarts
ensuring independence across lengths. Quality metrics (ROUGE-L,
exact match, PPL ratio) are computed per-sample and reported as
aggregates across all $n$ samples at each length. The number of
samples per length is specified in each table's caption.

confidence intervals} ($n{=}12$, 10{,}000 bootstrap iterations),
summarised in Table~\ref{tab:bootstrap-ci}.

Crucially, \semblend demonstrates both quality preservation and

\subsubsection{SemBlend Injection: Scaling Behavior}
\label{sec:semblend-injection-results}

The monotonically increasing speedup in Table~\ref{tab:results-kv}
(Figure~\ref{fig:speedup-curve}) reflects the key architectural property:
SemBlend TTFT is bounded by CPU$\to$GPU KV transfer bandwidth, growing
sub-linearly with prompt length, while cold prefill scales linearly.
Longer prompts benefit most from semantic KV reuse.

\paragraph{Why SemBlend complements prefix caching.}
vLLM's native prefix cache and LMCache's block-hash caching handle
exact token matches efficiently but \emph{fail entirely} for semantically
similar prompts with lexical variation. Importantly, both systems fail
for the same fundamental reason: any change in instruction prefix
length shifts \emph{all} 256-token chunk boundaries, invalidating every
chunk hash despite identical article content.

\semblend provides value when:
(1)~prompts share semantic content but differ lexically
(paraphrased queries, synonym substitution, reformulations, or
different instruction phrasings around the same document);
(2)~GPU KV cache is saturated and entries have been evicted to CPU;
(3)~prompts arrive at different vLLM replicas (prefix caches are
instance-local). In our benchmarks, \semblend correctly identifies
donors at cosine similarity $\geq 0.60$ and injects their KV via
LMCache chunk-swap, achieving $1.5\times$--$5.8\times$ hit-only speedup
on real NLP datasets at 2K--8K (Table~\ref{tab:cross-dataset}), and
$3.0\times$--$3.4\times$ on cross-instruction queries where LMCache
achieves no reuse at all.
The 16K cross-instruction scenario sustains 100\% hit rate and
$3.16\times$--$3.30\times$ speedup, confirming that sliding window
text sampling preserves embedding quality at longer contexts.

\paragraph{Component overhead.}
The \semblend pipeline adds ${\sim}8$\,ms decision overhead:
MiniLM-L6-v2 CPU embedding (${\sim}5$\,ms), numpy cosine search +
candidate ranking (${\sim}2$\,ms at $N{=}1$K; the cosine scan itself
is 0.04\,ms per the scaling table below, with the remainder from
candidate filtering and token-set alignment), rapidfuzz alignment
(${\sim}1$\,ms).
When a donor is found, LMCache's CPU$\to$GPU KV transfer takes
${\sim}35$\,ms for ${\sim}8$K tokens, dominated by PCIe bandwidth.
The 10\,GiB CPU buffer holds ${\sim}23$ donors at 8K tokens
(each 7B-AWQ donor with 4 GQA KV heads requires ${\sim}448$\,MiB).

\paragraph{Multi-candidate KV fallback.}
A key challenge is the decoupling between SemBlend's donor store
(embedding-based, never evicts) and LMCache's KV buffer (LRU eviction
when CPU memory fills). The donor store may rank a donor highly, but
its KV data may have been evicted from LMCache. SemBlend addresses
this with multi-candidate fallback: \texttt{find\_donor\_candidates()}
returns ranked candidates, and the connector tries each one's KV in
LMCache until it finds one with available data. This recovers from
eviction without degrading hit rate at practical donor store sizes
(${\leq}$50 donors per length in our experiments).

chunk. PARAPHRASE and PARTIAL variations change tokens within chunks,

Table~\ref{tab:component-breakdown} decomposes the \semblend
pipeline into its constituent stages. The total pipeline overhead of
${\sim}8$\,ms is $20\times$ lower than a remote embedding gateway's

The dominant cost is MiniLM-L6-v2 embedding (${\sim}5$\,ms), which runs
on a single CPU core with no GPU contention. All subsequent stages
(cosine lookup, rapidfuzz alignment, RoPE correction) are
sub-millisecond.

\paragraph{Finding 1: In-process embedding eliminates the overhead
bottleneck.}
A remote embedding gateway (${\sim}150$\,ms) would dominate the
donor discovery path, making the system viable only for models
$\geq$7B at $\geq$2K tokens. \semblend's in-process MiniLM
(${\sim}5$\,ms) reduces total pipeline overhead to ${\sim}8$\,ms,
widening the operating envelope to \emph{any model at any prompt
length where cold prefill exceeds ${\sim}50$\,ms}.

\paragraph{Finding 2: Embedding vs.\ Jaccard trade-offs.}
An ablation replacing MiniLM embedding with token-set Jaccard
similarity shows complementary strengths: Jaccard achieves 99--100\%
donor match rate for cluster variations (REORDER, PARAPHRASE) where
token overlap is high, while MiniLM embedding achieves 7\% for the
same clusters---primarily because MiniLM-L6-v2 processes only the
first 512 tokens of an 8K-token prompt, losing semantic information
distributed across the full document. However, Jaccard is $O(T)$
per candidate and fails for true paraphrase (low token overlap despite
semantic equivalence). \semblend uses embedding cosine as the
primary discovery signal for its $O(1)$ lookup per candidate and
coverage of semantic paraphrase; the practical cross-dataset
hit rates (62--100\%) confirm that MiniLM's 512-token window is
sufficient for detecting similarity between variations of the same
source article---the dominant use case in RAG workloads where the
same context document is queried repeatedly with different formulations.

\paragraph{Finding 3: RoPE correction produces near-zero per-layer deviation.}
Token-level alignment uses rapidfuzz~\cite{rapidfuzz} (compiled C++
Levenshtein) for fine-grained matching and chunk-level content-hash
alignment for REORDER detection (95.3\% vs.\ 28.5\% reuse ratio).
RoPE delta correction (\S\ref{sec:rope-correction}) solves position
encoding at $O(Nd)$ cost per layer (${\sim}7\,\mu$s for 8K tokens
on A10G).

\paragraph{Bathtub curve calibration.}
We empirically calibrate the bathtub curve directly from the live
Qwen2.5-7B-AWQ deployment (28 layers, A10G)
by logging per-layer K-vector norms across 30 semantically similar
prompt pairs. The measured norm profile follows the expected funnel
pattern: $\|K\|_2 \approx 134\text{K}$ at layer~0,
$\approx 10\text{K}$ at middle layers, rising to $\approx 207\text{K}$
at layer~27. Fitting the bathtub model (Eq.~\ref{eq:bathtub}) to
these norms yields $\sigma_e{=}0.054$, $\sigma_l{=}0.126$,
$\beta{=}3.94$ ($R^2{=}0.76$), confirming the funnel shape with
elevated late-layer sensitivity. The measured $\sigma_l {>} \sigma_e$
confirms the same \emph{directional} asymmetry as the Qwen preset
($\sigma_l{=}0.35 {>} \sigma_e{=}0.15$), validating the
architecture-specific calibration direction. The absolute magnitudes
differ ($2.7\times$ smaller), likely because the deployment presets
include a safety margin; the empirical profile confirms that late
layers are more sensitive than early layers for Qwen, as the preset
assumes.

A complementary experiment measures per-layer K-deviation between
donor and query pairs. With RoPE delta correction enabled, all
per-layer deviations are \textbf{below 0.2\%} for Qwen2.5-7B (peak:
0.06\% at layer 9) and \textbf{below 0.15\%} for LLaMA-3.1-8B
(peak: 0.15\% at layer 13). The deviation profile is essentially flat
rather than bathtub-shaped, indicating that RoPE correction eliminates
position encoding drift.

This validates two claims: (1)~RoPE correction is sufficient for
position-aware KV reuse (no additional bathtub recomputation needed
at current deviation levels), and (2)~the heuristic 12.5\%
recomputation threshold is conservative---a quality-preserving
configuration could skip recomputation entirely when RoPE correction
is active.

A complementary behavioral validation confirms the bathtub layer
selection is sufficient for output quality: running REORDER
variations (the canonical layer-scope stress test) through the
current bathtub configuration, we measure ROUGE-L$=1.000$ on hit
pairs---output quality on bathtub-recomputed hits is identical to
cold output, confirming that the selected layers fully capture the
information needed for faithful generation.

\paragraph{Prefill computation savings.}
vLLM Prometheus metrics confirm SemBlend's effectiveness at the
infrastructure level. The external prefix cache (LMCache/SemBlend)
\textbf{eliminates prefill computation for cached tokens}, as vLLM
skips attention over tokens whose KV blocks are already populated.
On Qwen2.5-7B-AWQ, the external prefix cache achieves a \textbf{74.3\%}
token hit rate (766K of 1.03M queried tokens)---meaning 74.3\% of
attention computation is avoided across the benchmark. vLLM's internal
prefix cache contributes only 0.02\% (208 tokens). On LLaMA-3.1-8B-AWQ,
external hits reach \textbf{52.1\%} (724K of 1.39M tokens) with
internal prefix cache at 13.8\% (192K tokens). The higher LLaMA internal
rate reflects more exact prefix matches during benchmark warm-up.
The $>$50\% token-level computation saving across both models corresponds
directly to the observed TTFT speedups: every cached token avoids its
share of the $O(N^2 d)$ attention computation that would otherwise
dominate cold prefill.

\subsection{Operating Envelope}
\label{sec:operating-envelope}

\paragraph{Expected TTFT as a function of hit rate.}
Under independence between hit outcome and prompt content,
the expected TTFT decomposes as:
\begin{equation}
  \mathbb{E}[\text{TTFT}] = P_h \cdot \text{TTFT}_\text{hit}
    + (1 - P_h) \cdot \text{TTFT}_\text{cold}
  \label{eq:expected-ttft}
\end{equation}
where $P_h$ is the hit rate and TTFT$_\text{hit}$, TTFT$_\text{cold}$
are the conditional means.
We validate this model empirically at 8K tokens on XSum
by running round-robin TTFT measurements at five hit fractions
$P_h \in \{0, 0.25, 0.50, 0.75, 1.0\}$ using fresh, previously-unseen
prompts per round to prevent LMCache contamination.

\emph{Empirical values:} TTFT$_\text{cold} = 3{,}428$\,ms,
TTFT$_\text{hit} = 303$\,ms, giving a $\mathbf{11.3\times}$ hit speedup.
The linear model (Eq.~\ref{eq:expected-ttft}) predicts within ${<}5\%$
of the measured E[TTFT] at all five fractions:

\small
\begin{tabular}{@{}rrr@{}}
\toprule
$P_h$ & Predicted (ms) & Measured (ms) \\
\midrule
0.00 & 3428 & 3428 \\
0.25 & 2638 & 2580 \\
0.50 & 1866 & 1843 \\
0.75 & 1114 & 1109 \\
1.00 & 303 & 303 \\
\bottomrule
\end{tabular}
\end{center}

\paragraph{Break-even analysis.}
The \semblend\ pipeline adds overhead on misses (embedding $+$ vector search
$+$ chunk alignment). We measure this empirically using unique-topic
prompts processed sequentially (cold$\to$hit$\to$miss per prompt, fresh
vLLM instance per context length):

\begin{tabular}{@{}rrrrrr@{}}

\noindent At 2K the pipeline overhead is effectively \emph{zero}:
the ${\sim}8$\,ms pipeline computation overlaps with the
$1{,}187$\,ms prefill compute window, adding no observable
wall-clock latency. Miss overhead grows
with context length (5\,ms at 4K, 212\,ms at 16K) because larger stores
require more chunk alignment checks; at steady-state sizes ($\sim$500
donors) it reaches ${\approx}496$\,ms.
Let $P_h^* = O_{\mathrm{miss}} / (O_{\mathrm{miss}} + S_{\mathrm{hit}})$
denote the break-even hit rate, where $S_{\mathrm{hit}}{=}\text{Cold}{-}\text{Hit}$.
\semblend\ is net-positive at \textbf{all tested context lengths}
($P_h^*{<}5\%$ at 2--16K tokens).
Even at steady-state store overhead, 8K requires only $P_h^*{=}17\%$;
at 32K (extrapolated), $P_h^*{=}3.4\%$.

\paragraph{Temperature invariance.}
KV injection is a prefill-side operation: sampling temperature does not
affect \semblend's hit rate or TTFT, confirmed empirically across
$T \in \{0.3, 0.7, 1.0\}$ ($\pm 1.5\%$ variance).

\paragraph{Threshold analysis.}
Three operating regimes emerge: \textbf{Conservative} ($\tau{\geq}0.80$)
accepts only exact and reorder types (38\% coverage, PPL 1.019).
\textbf{Balanced} ($\tau{=}0.60$, production default) is
\emph{Pareto-optimal}: lowering to 0.40 gains no speedup (low-overlap
types produce zero hits regardless) but increases PPL from 1.059 to 1.117.
\textbf{Aggressive} ($\tau{\leq}0.40$) accepts diverse (cosine 0.62,
PPL 1.249) and partial-40 (cosine 0.52, PPL 1.408) types with 11.7\%
PPL degradation and zero speedup benefit. The 0.60 default is thus
an empirically-calibrated Pareto operating point.

\paragraph{Empirical threshold sweep on live endpoint.}
A live endpoint sweep ($\tau \in \{0.30, 0.40, 0.50, 0.60, 0.70, 0.80\}$,
Qwen2.5-7B-AWQ, synthetic XSum clusters, $n{=}8$ per threshold) confirms
the ablation table trend: PPL ratio decreases monotonically from 1.21
($\tau{=}0.30$, 88\% hit) to 1.05 ($\tau{=}0.80$, 41\% hit), with
$\tau{=}0.60$ as the Pareto optimum (64\% hit, PPL 1.08). A separate
sweep on CNN/DailyMail 8K ($\tau \in \{0.50, 0.70, 0.80\}$, $n{=}8$)
yielded \emph{identical} PPL ratios (1.033) at all three thresholds.
This occurs because all CNN/DailyMail cluster pairs have measured cosine
similarity ${\geq}0.80$ by construction (semantically similar articles
from the same news cycle), so the cosine gate is not the active quality
filter in this regime.
The \emph{primary} quality gate is LMCache's 256-token block hash
matching: only blocks with identical token sequences are injected,
regardless of document-level cosine. One persistent quality outlier
(PPL ratio 1.270, speedup $1.34\times$) survives all thresholds
including $\tau{=}0.80$: the matched article covers the same news story
(cosine ${\approx}0.85$) but the donor's 256-token block was at a
\emph{different semantic position} in the article (incident description
vs.\ institutional response). This illustrates the \emph{semantic
staleness} failure mode~\cite{prophetkv}: block-level hash
identity does not guarantee semantic position alignment.

\paragraph{Context gate: eliminating isolated chunk matches.}
We address the semantic staleness failure mode with a
\emph{context gate} in the alignment layer: a chunk hash match is
accepted only if at least one adjacent target chunk also matches.
Isolated matches---where a single 256-token block is token-identical
but surrounded by unrelated content---are rejected because they
indicate positional displacement (same paragraph, different article
position). The gate adds zero measurable overhead (set intersection
on ${\sim}32$ chunk indices) and eliminates the FM2-class outlier by
construction: the PPL=1.27 case had exactly one isolated matching
chunk whose semantic context was incompatible. The context gate is
enabled by default (\texttt{SEMBLEND\_CONTEXT\_GATE=1}) and can be
disabled for backward compatibility.

\paragraph{Quality invariance across variation types.}
To quantify quality across the full similarity gradient, we ran
a controlled experiment on 8 CNN/DailyMail 8K clusters, each with
7 variation types spanning cosine similarity 1.0 (exact copy) to
0.20 (topically diverse). Cold baselines were measured \emph{before}
donor registration to prevent vLLM's internal prefix cache from
contaminating cold TTFT measurements. Across all 56 variation-cluster
pairs per model, PPL ratio was $1.000 \pm 0.001$ on \textbf{both}
Qwen2.5-7B-AWQ and LLaMA-3.1-8B-AWQ---indistinguishable from cold
across all similarity levels (Figure~\ref{fig:variation-sensitivity}).
This follows directly from SemBlend's block-level quality gate:
accepted hits use verified exact-match KV blocks (inherently near-zero
quality loss), while rejected queries fall back to standard cold
inference (PPL ratio $\equiv 1.0$ by construction). The system
thus provides a \emph{quality floor guarantee}: no input can experience
worse quality than the cold baseline, regardless of model architecture.
The non-monotonic hit-rate pattern in
Figure~\ref{fig:variation-sensitivity}(b) confirms that
LMCache block-hash matching---not cosine similarity---is the operative
quality gate: partial-document variations (40\% overlap, cosine
${\approx}0.47$) achieve 62\% hit rate by sharing verbatim 256-token
paragraph blocks, while exact-copy variations (cosine $=1.0$) miss
due to minor tokenization differences.

Jaccard's 3/7. The two additional types (partial-80, partial-60) have
40--80\% token overlap and produce $1.5$--$5.6\times$ hit-only speedup
at 8K+ tokens (Table~\ref{tab:cross-dataset}). Paraphrase Jaccard is high
(0.985) because CNN/DailyMail synonym substitution replaces few words;
with LLM-rewritten paraphrases, Jaccard would drop below 0.60 while
cosine remains above. \semblend uses embedding cosine as primary
matching signal with Jaccard as $O(n)$ pre-filter for alignment gating.}
\end{table}

MiniLM cosine identifies partial-80 and partial-60 as viable donors
(cosine 0.80 and 0.65, respectively) while Jaccard rejects them (0.41
and 0.28, below 0.60). These partial-overlap types contribute
$1.5$--$5.6\times$ speedup at 8K+ tokens, making the 5/7 vs.\ 3/7
difference operationally significant. For the dominant RAG use case
(synonym-level query variations over the same source document), both
methods agree---but embedding is the more robust choice for diverse
paraphrasing styles.

\subsubsection{LMCache Chunk Size Ablation}
\label{sec:analysis:chunk-size}

A natural question is whether smaller LMCache chunks (default 256 tokens)
improve cross-instruction reuse by reducing boundary sensitivity.
We evaluate live E2E inference at chunk sizes
$\{64, 128, 256, 512\}$, deploying a separate vLLM+LMCache instance for
each configuration. Each deployment uses 1,000 cross-instruction pairs
(250 articles $\times$ 4 instruction variants, XSum text with synthetic
filler) at 8K tokens, measuring TTFT speedup and hit rate.

Contrary to offline alignment predictions (which suggested $2.5\times$

same length as the donor instruction, \emph{all} subsequent chunk boundaries
align and LMCache achieves full reuse; if the token count differs by even
one token, \emph{all} boundaries shift and no chunks match.
This binary alignment effect dominates chunk granularity, making chunk
size irrelevant for cross-instruction reuse in practice. The overall
speedup of $0.88\times$ reflects the net effect when 40\% of queries
are misses (misaligned instructions): the ${\sim}8$\,ms pipeline
overhead on each of the 400 misses outweighs the TTFT savings on the
600 hits.  Note that this benchmark isolates
\emph{cross-instruction} reuse; the primary use case of
same-instruction, semantically-similar content
(Table~\ref{tab:results-kv}) achieves 88--100\% hit rates and
$2.2$--$12\times$ speedup. Cross-instruction reuse is an additional
capability, not the core value proposition.

\subsubsection{Shifted-Prefix Limitation: PartialAttention Opportunity}
\label{sec:analysis:partial-attn-gap}

The chunk-swap injection path (\S\ref{sec:design}) guarantees $\Delta{=}0$
for all matched positions, making RoPE correction a no-op. This design is
correct but sacrifices reuse in \emph{shifted-prefix} scenarios where
identical document tokens appear at different absolute positions due to
varying instruction lengths, headers, or system prompt padding.

We quantify the gap with offline alignment on $n{=}300$ pairs at 8K tokens
across three scenarios (instruction-shift, header-shift, padding-shift):

\begin{itemize}[nosep]
\item \textbf{Chunk-level reuse: 0.0\%} --- every 256-token chunk boundary
  is misaligned by the prefix shift, so no hash matches occur.
\item \textbf{Token-level reuse: 99.1\%} (Levenshtein alignment) --- the
  document tokens are identical, just shifted.
\item \textbf{Recoverable compute: 99.1\%} of prefill tokens could be
  reused by a PartialAttention path with $\Delta{\neq}0$ RoPE correction.
\end{itemize}

This 99.1\% gap motivates the PartialAttention integration path
(\S\ref{sec:triton}): sub-chunk injection with RoPE delta correction
would recover nearly all shifted-prefix reuse. The infrastructure
(Triton kernels, RoPE correction) is implemented and validated
(\S\ref{sec:analysis:rope}); integration into vLLM's prefix-match
assumption is the remaining engineering step.

\subsubsection{RoPE Correction: Position-Shift Validation}
\label{sec:analysis:rope}

\paragraph{Quality invariance under RoPE correction.}
The variation sensitivity experiment (\S\ref{sec:analysis})
shows PPL ratio $= 1.000 \pm 0.001$ across all 7 variation types on both
models, including REORDER (50\% hit rate on Qwen, 62.5\% on LLaMA).
REORDER hit rate arises from shared prompt structure: prompts share a
common prefix (system prompt and early context), whose 256-token
LMCache blocks match between donor and REORDER variant at identical
positions ($\Delta{=}0$). For these shared-prefix blocks, RoPE
correction is a no-op (identity map), and quality is preserved.
Content blocks with reordered sentences do not match at the same
chunk boundaries (minor whitespace differences produce distinct hashes),
so the content portion falls back to cold inference (PPL $=1.0$ by
definition). In all cases PPL $\approx 1.0$, confirming the quality
floor guarantee (\S\ref{sec:analysis}): any miss falls back to cold
prefill; any hit reuses exactly-matched KV blocks with or without
RoPE correction depending on $\Delta$.
\paragraph{End-to-end $\Delta{\neq}0$ validation.}
To validate RoPE correction under realistic deployment conditions, we
inject artificial position offsets $\Delta \in \{256, 1024\}$ into
loaded donor K via the \texttt{FORCE\_DELTA} mechanism, which applies
$\mathrm{RoPE}(\Delta)$ to all matched K positions after LMCache load.
We compare three conditions per $\Delta$ across 10 diverse documents
(380--430 tokens each, cross-topic):

\begin{center}
\small
\begin{tabular}{@{}lccc@{}}
\toprule
\textbf{Condition} & \textbf{Avg PPL} & \textbf{Ratio} & \textbf{Pass} \\
\midrule
Baseline ($\Delta{=}0$) & 1.249 & 1.000$\times$ & REF \\
Uncorrected $\Delta{=}256$ & 1.985 & 1.589$\times$ & \checkmark \\
Uncorrected $\Delta{=}1024$ & 1.871 & 1.498$\times$ & \checkmark \\
Corrected $\Delta{=}256$ & 1.249 & 1.000$\times$ & \checkmark \\
Corrected $\Delta{=}1024$ & 1.247 & 0.998$\times$ & \checkmark \\
\bottomrule
\end{tabular}
\end{center}

\noindent
Uncorrected injection produces garbled output (Chinese characters,
random number sequences, repetitive fragments) with 1.50--1.59$\times$
PPL degradation. Corrected injection recovers \emph{identical} output
to baseline at both $\Delta$ values (PPL ratio $< 1.002$, 100\%
text-level recovery). Note: the FORCE\_DELTA test mechanism applies RoPE correction to
\emph{all} matched K positions across all 28 layers (a worst-case
stress test), taking ${\sim}2.7$\,s. In normal SemBlend operation,
RoPE correction is $O(Nd)$ per layer---${\sim}7\,\mu$s for 8K
tokens (\S\ref{sec:rope-correction})---and applies only to
positions with $\Delta{\neq}0$. The current chunk-swap path
guarantees $\Delta{=}0$, so RoPE correction is a no-op in
production; the FORCE\_DELTA test validates the kernel for
future PartialAttention paths where $\Delta{\neq}0$ will occur.

\subsubsection{When Competitors Are Better}

\semblend is not universally optimal. Some configurations yield
PPL ratio $<$1.0 (beneficial KV injection, observed in our data:
Qwen CNN/DM@2K=0.960, LLaMA WikiHow@2K=0.992), indicating that semantically related context occasionally
improves output quality---consistent with findings in prior semantic
KV work. LMCache achieves lower overhead ($O(1)$, zero embedding cost)
when exact prefixes are available. For workloads dominated by exact prefix sharing (e.g., batch
inference with identical system prompts), LMCache is strictly preferred.
\semblend's advantage is in the \emph{semantic gap}: queries that are
semantically related but lexically different, where LMCache provides
no cache benefit.

\subsubsection{Real-World Conversational Workload Validation}
\label{sec:analysis:wildchat}

To validate that the semantic similarity assumption holds beyond
synthetic benchmarks, we analyze WildChat-1M~\cite{zhao2024wildchat},
a large-scale dataset of 1 million real ChatGPT conversations. We
sample 100K rows, extract 33,160 consecutive conversation pairs from
5,000 users with $\geq$2 conversations each, and compute MiniLM-L6-v2
cosine similarity between each consecutive donor--query pair
(using the same model \semblend deploys at inference time).

\textbf{46.5\%} of consecutive conversation pairs from the same user
share cosine similarity $\geq 0.60$ (\semblend's production threshold),
with a median consecutive similarity of \textbf{0.565}
versus \textbf{0.073} for random cross-user pairs---a lift of
$\boldsymbol{+0.492}$ ($7.7\times$ higher). At the cluster level,
23.8\% of pairs exceed $\tau{=}0.80$ and 7.1\% exceed $\tau{=}0.95$
(near-exact repeats). The distribution is broadly bimodal: many users
either return to closely related topics (0.60--0.95) or ask
near-identical questions across sessions (0.95--1.00).

These results confirm that real-world chatbot workloads exhibit
substantial intra-user semantic correlation: nearly half of all
same-user consecutive conversation pairs would benefit from
\semblend's semantic KV reuse at the production threshold,
with 7.1\% eligible for exact-prefix LMCache reuse as well.
The $7.7\times$ lift over random cross-user pairs validates the
per-tenant deployment model: semantic donor caches should be
partitioned by user, not shared globally
(Figure~\ref{fig:wildchat-hist}).

enables \emph{zero-configuration} cross-tenant acceleration: no per-tenant
routing or cache partitioning is required.

\section{Discussion}
\label{sec:discussion}

\paragraph{Deployment boundaries.}
\semblend's in-process architecture adds only ${\sim}8$\,ms overhead,
making semantic KV reuse net-beneficial whenever cold prefill TTFT exceeds
${\sim}50$\,ms---which includes most models at ${\geq}256$ tokens
(compared to ${\sim}150$\,ms for a remote embedding gateway, which
would require ${\geq}2$K tokens on ${\geq}7$B models to break even).
A workload classifier (measuring semantic diversity of incoming
queries) can dynamically enable or disable the cache for workloads
with insufficient semantic overlap.

\paragraph{Relationship to exact-match systems.}
\semblend is complementary to LMCache and KV-aware routing systems
(e.g., NVIDIA Dynamo~\cite{dynamo}), not a replacement. LMCache handles
exact prefix matches with $O(1)$ token-hash lookup and zero quality risk;
\semblend activates only when the exact-match path misses and embedding
similarity exceeds the threshold. KV-aware routers increase fleet-wide
cache hit rates by directing requests to instances that already have
the relevant exact prefix warm---but routing to the ``most similar''
instance does not produce a cache hit if the tokens differ. \semblend
provides the last-mile KV reuse for the cases routing cannot solve:
semantically similar prompts with different wording where no instance
has an exact prefix match. The three approaches are thus complementary:
routing for exact-match coverage at scale; LMCache for exact-match KV
transfer; \semblend for semantic-variation coverage. The optional CAGRA tier
(\S\ref{sec:scaled}) fills the routing gap at the fleet level: embedding
similarity selects the target instance (providing the same benefit as
semantic routing), while KV injection with RoPE correction enables reuse
once the request arrives.

\paragraph{PartialAttention integration.}
The three Triton CUDA kernels (\S\ref{sec:triton}) integrate with vLLM's
model runner via a \texttt{patch\_model\_runner()} hook
that intercepts prefill when a PartialAttention plan is active. \textbf{RoPE delta correction}
(\S\ref{sec:rope-correction}) enables deployment on all RoPE-based
models: after donor K is scattered to target positions, the correction
kernel applies $\text{RoPE}(\Delta)$ in-place at ${\sim}7\,\mu$s
per layer for 8K tokens---negligible compared to the attention
computation being avoided. The bathtub curve (\S\ref{sec:recomp})
complements this by selectively recomputing early/late layers where
attention drift dominates, while middle layers use RoPE-corrected
donor KV.

\paragraph{Ablation: component contributions.}
We isolate two key components via ablation on Qwen2.5-7B-AWQ
synthetic benchmarks at 8K tokens:

\textbf{(1)~PartialAttention (bathtub recomputation):}
Disabling selective layer recomputation yields $5.51\times$ speedup
(vs.\ $5.66\times$ with recomputation)---a marginal 2.6\% difference---while
PPL ratios remain acceptable (0.964--1.018). The primary speedup
derives from KV transfer bypassing full prefill, not recomputation
savings. PartialAttention's value is in \emph{quality insurance}
for high-drift layers and can be disabled for throughput-focused
workloads.

\textbf{(2)~RoPE delta correction --- scope and validation:}
RoPE correction activates when matched KV chunks land at different
absolute positions between donor and target sequences
($\Delta = t_{\text{pos}} - d_{\text{pos}} \neq 0$).
In the current 256-token LMCache chunk deployment, paragraph-level reordering
aligns KV reuse to chunk boundaries so $\Delta{=}0$ for matched pairs,
making correction a no-op at negligible overhead (${\sim}7\,\mu$s/layer).
End-to-end validation with artificially injected $\Delta \in \{256, 1024\}$
confirms the correction is essential: uncorrected reuse degrades PPL by
1.50--1.59$\times$ (garbled output), while corrected reuse recovers
baseline quality exactly (PPL ratio $< 1.002$) across 10 diverse documents
(\S\ref{sec:analysis:rope}).
Correction is enabled by default as forward-compatible infrastructure
for sub-chunk or sentence-level injection where $\Delta{\neq}0$.

\paragraph{Threats to validity.}
(1)~KV-tensor speedups are measured with streaming TTFT on a
single-request workload; throughput measurements at $c{=}1$--$32$
confirm zero degradation (9+ QPS, zero errors) on short prompts,
and long-context throughput (8K XSum, $c{=}1$--$8$) achieves
14+ QPS with zero errors, confirming donor reuse at scale.
(2)~The quality model parameters are fit against published aggregate
metrics rather than per-token quality measurements. Empirical output
quality verification (Table~\ref{tab:quality}) addresses this for
the scenarios we evaluate.
(3)~\semblend uses MiniLM-L6-v2 (384-dim) which has lower
embedding quality than jina-v4 (1024-dim). An embedding ablation
(Table~\ref{tab:embedding-ablation} in \S\ref{sec:analysis}) shows that
MiniLM cosine accepts 5/7 variation types at $\tau{=}0.60$ vs.\
Jaccard's 3/7; critically, MiniLM captures partial-80 and partial-60
types that Jaccard misses. For high-overlap variations (exact, reorder,
paraphrase), both methods agree. In practice, 384-dim embeddings
are sufficient for nearest-neighbor donor discovery in RAG workloads
where the same document is queried with variations.
(4)~Single-GPU experiments; multi-GPU tensor-parallel serving with
sharded KV caches may change the overhead profile.
(5)~KV-tensor results use $n{=}5$--$8$ per prompt length with per-length
server restarts, providing clean measurements but limiting total sample
size per configuration.

\subsubsection{Multi-Turn Dialogue}
\label{sec:multiturn}

Multi-turn conversations are a natural fit for KV cache reuse: each
successive turn shares most of the prior context. We evaluate 200
independent conversations at 4K tokens, measuring TTFT across 3 turns
(turn~1 establishes the cold baseline, turns 2--3 append user/assistant
content and re-query).

\begin{center}
\small
\begin{tabular}{@{}rrrr@{}}
\toprule
\textbf{Turn} & \textbf{P50 TTFT (ms)} & \textbf{Hit Rate} & \textbf{Speedup} \\
\midrule
1 (cold)  & 4,019 & ---    & $1.0\times$ \\
2         & 791   & 99.5\% & $5.08\times$ \\
3         & 787   & 99.5\% & $5.10\times$ \\
\bottomrule
\end{tabular}
\end{center}

Turn~2 achieves $\mathbf{5.1\times}$ speedup with 99.5\% hit rate
(199/200 conversations). The near-perfect hit rate is expected:
multi-turn prompts extend the same prefix, producing identical
LMCache chunk alignment. The single miss is attributable to LRU
eviction under the 15\,GB CPU buffer constraint. This result
validates \semblend as effective for the common multi-turn chat
serving pattern.

\subsubsection{Code Generation (Negative Control)}
\label{sec:codegen}

To measure overhead on workloads with \emph{no} semantic overlap,
we evaluate 1,000 independent coding problems at 4K tokens.
Each prompt contains a unique algorithmic task with no shared
context between problems.

\textbf{Result:} 0.0\% hit rate, $0.96\times$ speedup (4\% overhead).
SemBlend correctly determines that no donors match (cosine similarity
$< \tau{=}0.60$) and falls through to cold prefill. The 4\% overhead
comprises the MiniLM embedding (${\sim}5$\,ms) and cosine search
(${\sim}1$\,ms) on every request. This confirms that \semblend
imposes minimal interference on dissimilar workloads.

\subsection{Limitations}\label{sec:limitations}
\textbf{Shifted-prefix gap.}
The chunk-swap injection path guarantees $\Delta{=}0$ but cannot reuse
tokens when instruction prefixes differ in length, shifting all
subsequent chunk boundaries. Offline analysis
(\S\ref{sec:analysis:partial-attn-gap}) shows 0\% chunk-level reuse
vs.\ 99.1\% token-level reuse in shifted-prefix scenarios at 8K tokens---a
quantified opportunity for the PartialAttention path with RoPE $\Delta{\neq}0$
correction, which is validated end-to-end (\S\ref{sec:analysis:rope}).
\textbf{Attention drift.}
While RoPE delta correction solves position encoding exactly, attention
pattern drift from context differences is addressed heuristically by
the bathtub curve; a more principled per-request calibration could
further improve quality. Bathtub presets are calibrated for Qwen2.5-7B
and LLaMA-3.1-8B; GQA, MoE, and very deep models ($>$80 layers) may
require recalibration.
\textbf{Fleet routing.}
Cross-instance donor discovery via the optional CAGRA tier is validated on
a controlled 8-worker fleet (\S\ref{sec:scaled}), achieving $2.6$--$3.8\times$
speedup; routing accuracy on diverse production workloads will depend on
the semantic overlap distribution.
\textbf{Short prompts} ($<$500 tokens) may not benefit, as the
${\sim}8$\,ms pipeline overhead can exceed the cold prefill time saved.
\textbf{Sample sizes.} Cross-dataset experiments use $n{=}5$--$8$
per condition; larger benchmarks use $n{=}150$--$1{,}000$ (WildChat,
multi-turn dialogue, code generation). While
Table~\ref{tab:bootstrap-ci} and Table~\ref{tab:wildchat-e2e} provide
95\% bootstrap CIs, the smaller cross-dataset experiments would benefit
from larger $n$, particularly for conditions with $<$100\% hit rate.
\textbf{No head-to-head baselines.} We compare against cold prefill
and LMCache prefix caching (Table~\ref{tab:prefix-cache-gap}), but
do not benchmark against CacheBlend~\cite{cacheblend} or
KVShare~\cite{kvshare} directly, as neither is publicly available
in a vLLM-integrated form. The taxonomy
(Table~\ref{tab:taxonomy}) provides qualitative comparison.
\textbf{Benchmark scope.} Evaluations span NLP summarization
datasets (CNN/DM, XSum, WikiHow, SAMSum, MultiNews), WildChat
conversations, multi-turn dialogue (\S\ref{sec:multiturn}), code
generation (\S\ref{sec:codegen}), and enterprise workloads
(\S\ref{sec:enterprise}). Larger-scale production deployments with
diverse query distributions would further characterize the
operational envelope.

\subsection{Future Work}\label{sec:future-work}

\paragraph{Sub-chunk KV injection.}
Current chunk-swap injection requires aligned 256-token boundaries,
losing all reuse when instruction-length differences shift chunks
(0\% chunk hit vs.\ 99\% token hit; \S\ref{sec:analysis:partial-attn-gap}).
Integrating the validated PartialAttention kernels (\S\ref{sec:triton})
with RoPE $\Delta{\neq}0$ correction would enable arbitrary
position injection, recovering this quantified opportunity.

\paragraph{Multi-GPU and tensor-parallel serving.}
All experiments use single-GPU (A10G) inference. Tensor-parallel
deployments shard KV across devices, requiring coordinated chunk-swap
on each shard. Extending LMCache's CPU offload to shard-aware transfer
is a natural next step.

\paragraph{Adaptive threshold and embedder selection.}
The fixed $\tau{=}0.60$ threshold and MiniLM embedder are validated
across our benchmarks but may not be optimal for all domains.
A workload-adaptive threshold---adjusting $\tau$ based on observed
hit-rate and quality feedback---could improve the precision--recall
trade-off. Hybrid embedder strategies (Jaccard for high-overlap
variations, MiniLM for paraphrase detection) could further improve
donor match quality.

\paragraph{Production deployment studies.}
Our WildChat analysis (\S\ref{sec:analysis:wildchat}) establishes real-world
semantic overlap, but production validation at scale---with diverse
user populations, query distributions, and SLA requirements---remains
future work.

\section{Conclusion}
\label{sec:conclusion}

We presented \semblend, a semantic KV cache system that extends
LMCache's exact-prefix matching to semantic donor discovery via an
\textbf{in-process architecture}. \semblend performs
donor discovery in ${\sim}8$\,ms (MiniLM CPU embedding +
numpy cosine + rapidfuzz alignment + calibrated bathtub) with zero
network hops. For deployments exceeding 100K donors, the optional
GPU-accelerated CAGRA tier provides cross-instance discovery at a
measured 1.37\,ms constant from $N{=}100$ to $N{=}1\text{M}$ donors (A10G).

At the KV-tensor level, we solve the Rotary Position Embedding (RoPE)
challenge that has constrained all prior semantic KV reuse systems to
prefix-only scenarios. \textbf{Exact RoPE delta correction}
(\S\ref{sec:rope-correction}) exploits the rotation group structure to
correct donor K tensors to arbitrary target positions at $O(Nd)$ cost
per layer---mathematically exact, with zero approximation error and
validated end-to-end: artificially injected $\Delta{\in}\{256,1024\}$
degrades uncorrected output by 1.5--1.6$\times$, while correction
recovers baseline quality exactly (PPL ratio $< 1.002$). Combined
with \textbf{calibrated bathtub recomputation} that addresses the
orthogonal problem of attention pattern drift, \semblend enables fully
non-contiguous KV reuse (REORDER, PARTIAL, paraphrase) on all
RoPE-based models.

Four takeaways stand out:

\textbf{(1) Semantic KV reuse works at scale.}
$\mathbf{2.2\times}$--$\mathbf{{\sim}12\times}$ TTFT
speedup across 2K--32K tokens on two architectures
(Figure~\ref{fig:speedup-curve}), with $1.97$--$3.78\times$ hit-only on
five real NLP datasets at 8K. Enterprise workloads confirm:
template reports ($4.24\times$, 100\% hit), customer support
($2.48\times$, 100\%), multi-tenant APIs ($3.89\times$ hit-only).
On 400 real WildChat pairs, ${\sim}30\%$ hit rate with
$1.63$--$1.88\times$ hit-only speedup.

\textbf{(2) Quality is preserved.}
On datasets with ${\geq}88\%$ hit rate (WikiHow, XSum, synthetic),
PPL ratio stays ${\leq}1.065$ on both models. Elevated PPL on
lower-hit datasets (SAMSum, CNN/DM) is driven by miss-run outliers,
not KV injection. The quality floor guarantee holds by construction:
accepted hits use verified exact-match KV blocks; rejected queries fall
back to cold prefill ($\text{PPL} \equiv 1.0$).

\textbf{(3) Architecture scales from in-process numpy to GPU ANN.}
In-process numpy handles up to 10K donors at ${\leq}0.14$\,ms;
the optional cuVS brute-force tier covers up to 100K at ${\leq}0.71$\,ms; CAGRA
ANN achieves $\mathbf{1.37}$\,ms \emph{constant} at $N{=}1\text{M}$
donors---in-process with no network hops on the critical path.

\textbf{(4) Architecture-specific bathtub improves reuse quality.}
Selectively recomputing high-drift layers (LLaMA: early layers $[0,1,2]$
at 9.4\%; Qwen: late layers $[26,27]$ at 7.1\%) addresses attention
pattern drift orthogonal to RoPE position correction, consistent with
architecture-specific layer criticality findings~\cite{kvphysics,droidspeak}.

\semblend is, to our knowledge, the first system combining exact RoPE
correction with calibrated layer-level recomputation to enable fully
non-contiguous semantic KV reuse on production transformers.
All code, datasets, and benchmark scripts will be released with the
camera-ready version; bootstrap confidence intervals use 10,000
resamples (\texttt{scipy.stats.bootstrap}).
